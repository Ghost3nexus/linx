4. 新規顧客フロー（体験→入会）

  1. LINEで体験申込 — お客様がLINE公式アカウントに「体験したい」と送信。AIが空き日時を提示し、お客様が選択すると予約確定。
  2. スタッフ通知 — 予約が入るとスタッフのLINEに自動通知。ダッシュボードの予約管理にも反映。
  3. 体験実施 — スタッフが体験レッスンを実施。
  4. 「完了」にする — ダッシュボードの予約管理で該当予約をクリック → 「完了にする」ボタンを押す。
  5. 入会案内自動送信 — 完了にした瞬間に、お客様のLINEに入会案内メッセージと全プランのStripe決済リンクが自動送信される。
  6. 3日後フォローアップ — 未入会の場合、3日後に自動で「いかがでしたか？」のフォローアップメッセージが送信される。
  7. 管理画面からも可能 — 会員管理の💳ボタンからいつでも決済リンクを作成可能。
💡 料金プラン:
セミパーソナル（通い放題）: ¥19,800/月
マンツーマン: ¥34,800/月
入会金: 通常¥30,000 / キャンペーン（体験当日）¥10,000

5. 既存会員フロー

  1. 会員がLINEでレッスン予約 — 「明日の15時に予約したい」→ AIが空き確認→予約確定
  2. 来店時チェックイン — 会員管理ページで該当会員の✅ボタンを押す → 出席記録 + 来店回数が自動カウント
  3. ステータス管理 — アクティブ / 休会 / 退会 を会員カードの編集ボタンから変更

6. スケジュール管理

曜日ごとのレッスン枠を管理します。各枠に担当スタッフをアサインできます。

  • 曜日タブ: 月〜日の各曜日に今週の日付と祝日が表示される
  • カレンダー上でスタッフアサイン: シフト枠のドロップダウンから担当スタッフを選択
  • Copy Day: ある曜日のスケジュールを別の曜日にコピー
  • 臨時休業: 「臨時休業」ボタンから6ヶ月先まで日付を設定可能
⚠️ 日付は全てJST（日本標準時）で管理されます。祝祭日は赤字で表示されます。

7. スタッフ管理

サイドメニューの「スタッフ管理」から独立ページで管理します。

  • スタッフ追加: 名前・役職・メール・LINE User ID を登録
  • 基本シフト: 左のスタッフ一覧から選択 → 右側で曜日ごとの出勤時間を設定
  • 休み設定: 2ヶ月分のカレンダーで日付タップ → 休みON/OFF
  • LINE通知: LINE User IDを設定すると、予約が入った時にスタッフのLINEに自動通知

8. 予約管理（週間カレンダー）

Googleカレンダー風の週間ビューで予約を一覧管理します。

  • 横軸: 月〜日の7日間（日付・曜日・祝日表示）
  • 縦軸: 9:00〜22:00の時間帯
  • 予約ブロック: 確定=緑、完了=青で表示。クリックで詳細モーダル
  • ステータス変更: 詳細モーダルから「完了にする」「キャンセル」が可能
  • 手動予約追加: 「予約を追加」ボタンから管理画面で直接予約作成

10. 出席管理（チェックイン）

  1. 会員管理ページを開く
  2. 来店した会員の ✅ ボタンをタップ
  3. 出席記録が保存され、来店回数が+1される

出席履歴はAPIから取得可能で、将来のデータ分析基盤になります。

11. 決済（Stripe連携）

自動（体験完了後）

予約を「完了」にした瞬間に、LINEで全プランの決済リンクが自動送信されます。

手動（管理画面から）

  1. 会員管理 → 対象会員の💳ボタンをタップ
  2. プラン（セミパーソナル / マンツーマン）を選択
  3. 入会金（キャンペーン¥10,000 / 通常¥30,000）を選択
  4. 「決済リンクを作成」→ Stripe Checkout画面が開く
  5. お客様にURLを共有、またはその場で決済

12. LINE一斉配信

  1. 会員管理ページの「LINE配信」ボタンをタップ
  2. 配信先を選択（アクティブ会員 / 休会中 / 体験者）
  3. メッセージを入力
  4. 「送信」→ 対象のLINE User IDを持つ全会員に一斉送信
💡 活用例: キャンペーン告知、年末年始のスケジュール変更、新プログラム案内、休会者への復帰促進メッセージ

14. 既存会員の移行方法

既存の会員情報をLINXに移行する方法は3つあります。

方法A: CSV一括インポート（推奨）

  1. 既存の会員管理システムからCSVをエクスポート
  2. 会員管理ページの「CSV取込」ボタンをクリック
  3. CSVファイルをアップロード → 自動で会員登録

対応ヘッダー: 名前/氏名/会員名, メール, 電話/電話番号, プラン/会員ランク, ステータス, 備考/メモ, 会員ID/会員番号

方法B: LINE経由移行

  1. 既存のLINE公式アカウントで会員情報登録フォームのリンクを配信
  2. 会員が名前・会員番号・プランを入力
  3. LINE User IDと紐づいてLINXに自動登録

方法C: 段階移行

  1. 新規入会者はLINXで管理開始
  2. 既存会員は来店時にLINE友だち追加を案内
  3. 1ヶ月かけて全会員を移行
⚠️ 移行方法はGOOD LIFE GYM様の運用に合わせて選択してください。 全店舗一斉移行 or 店舗ごと段階移行など、ご要望に応じて対応します。

15. 臨時休業・祝日管理

  • 臨時休業: スケジュール画面の「臨時休業」ボタン → 6ヶ月先までカレンダーで日付タップ
  • 祝祭日: 2025-2027年の日本の祝日が自動表示（赤字）
  • スタッフ休み: スタッフ管理 → 2ヶ月分カレンダーで個別に設定

16. よくある質問

Q. AIが間違った回答をした場合は？

「お店の情報」ページでナレッジを追加・修正してください。AIはナレッジに基づいて回答します。

Q. AIが対応できない質問はどうなる？

AIが判断できない場合は「担当者に確認します」と回答し、管理者のLINEにエスカレーション通知が届きます。

Q. 予約がダッシュボードに反映されない

LINE経由の予約は通常即座に反映されます。反映されない場合はページを更新してください。

Q. スタッフに通知が届かない

スタッフ管理でLINE User IDが正しく設定されているか確認してください。

Q. 複数店舗で使える？

はい。アカウントごとに独立しているため、店舗ごとにアカウントを作成すれば複数店舗管理が可能です。

Q. 既存の公式LINEアカウントと併用できる？

はい。既存のLINE公式アカウントのWebhook URLをLINXに設定するだけで、既存の友だちリストがそのまま使えます。

17. 今後のロードマップ

LINXは継続的に機能拡張を予定しています。以下は今後の開発計画です。

✅ Phase 1（現在）— 基本機能

LINE予約・AI応対・スタッフ管理・会員管理・Stripe決済・チェックイン・LINE配信

📋 Phase 2 — 事前フォーム・提案書自動生成

無料体験前の事前フォーム入力: 体験予約確定後、LINEでヒアリングフォームを自動送信。目的（ダイエット/筋力UP/健康維持）、運動経験、持病・怪我の有無を事前収集。

体験後の提案書自動生成: 事前フォーム+体験時の体組成データを元に、AIがパーソナライズされた入会提案書（推奨プラン・トレーニングメニュー・目標設定）をダッシュボード上で自動生成。スタッフはワンクリックでお客様のLINEに送信。

📊 Phase 3 — 顧客分析・データダッシュボード

顧客分析: 来店頻度・継続率・離脱予測・LTV（顧客生涯価値）をグラフで可視化。休会予備軍を自動検出し、フォローアップをAIが提案。

売上分析: 月次MRR・プラン別比率・入会率・退会率をダッシュボードで一覧管理。

👥 Phase 4 — スタッフ稼働分析・AIシフト生成

スタッフ稼働分析: 担当レッスン数・出席率・顧客満足度をスタッフごとに可視化。

AIシフト自動生成: 月初にAIが翌月のシフト案を自動作成 → 各スタッフのLINEに配信 → 休み希望・変更希望をLINEで返信 → AIが自動調整・確定 → ダッシュボードに反映。スタッフ間のLINEやりとりだけでシフトが完成。

🏢 Phase 5 — マルチ店舗統合管理

本部ダッシュボード: 全FC店舗の予約数・売上・会員数を一元管理。店舗間のスタッフ移動・ヘルプ調整もシステム上で完結。

ベンチマーク: 店舗間のKPI比較（入会率・継続率・客単価）で優秀店舗のノウハウを横展開。

💡 開発優先度はGOOD LIFE GYM様のご要望に応じて調整します。各Phaseの詳細仕様・スケジュールは別途ご相談ください。
