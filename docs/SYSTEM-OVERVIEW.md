## 0. このドキュメントについて

### 目的
LINX を導入・運用する店舗オーナーおよび現場スタッフが、**最短ルートで AI コンシェルジュを立ち上げ、日々の予約・会員管理・顧客対応までを1画面で完結できるようになること**。本書は営業資料ではなく、実運用の手引書です。

### 対象者
1. **店舗オーナー / 経営者** — 導入判断・プラン選定・費用対効果の把握
2. **店長 / マネージャー** — 日常運用・スタッフ配置・会員管理
3. **現場スタッフ** — 予約受付・会員チェックイン・LINE対応
4. **本部担当者（FC展開時）** — 複数店舗の横断管理、店舗切替

**前提スキル**: LINE 公式アカウントの基本操作ができれば十分。プログラミング知識は不要。

## 1. LINX とは

**LINX（リンクス）** は、LINE 公式アカウントに AI コンシェルジュを追加する SaaS です。お客様からの LINE メッセージに AI が自動で応答し、そのまま予約・会員確認・決済まで完結させます。

### ひとことで
> **「公式LINEに、AIスタッフを。」**  
> 24時間対応の受付・予約受付担当を、月額固定費だけで雇う感覚。

### 解決する課題
| 従来の痛み | LINX 導入後 |
|---|---|
| 営業時間外の問い合わせを翌日対応 → 機会損失 | 24時間AIが回答、予約まで完結 |
| スタッフが予約電話で手が離せない | AIが空き枠を提示、即予約確定 |
| ノーショー（無断キャンセル）で収益ロス | 自動リマインド + No-show時の自動リスケ提案 |
| 予約管理システムが高い（hacomono等 月5万〜） | LINX 月1万円前後、機能は必要十分 |
| 複数システムを行き来（LINE・予約・会員・決済） | ダッシュボード1つに集約 |

## 3. システム全体像

```
   ┌─────────────┐       ┌────────────────┐      ┌────────────────┐
   │  お客様     │─LINE─▶│ LINE公式AC     │─────▶│ LINX API       │
   │ (スマホ)    │◀──────│ (店舗のもの)   │◀─────│ (Railway)      │
   └─────────────┘       └────────────────┘      └────────┬───────┘
                                                          │
                                                 ┌────────┴────────┐
                                                 │                 │
                                          ┌──────▼──────┐  ┌───────▼──────┐
                                          │ Claude AI   │  │ PostgreSQL   │
                                          │ (応答生成)  │  │ (DB)         │
                                          └─────────────┘  └──────────────┘
                                                                 │
   ┌─────────────┐       ┌────────────────┐                     │
   │ 店舗スタッフ│───────▶│ ダッシュボード │◀────────────────────┘
   │             │       │ (Vercel)       │
   └─────────────┘       └────────────────┘
```

### 構成要素
- **LINE 公式アカウント** — お客様との接点。店舗が所有
- **LINX API サーバー** — 応答生成・DB・決済・予約ロジック（Railway）
- **Claude Sonnet** — 自然言語理解と応答生成
- **PostgreSQL** — 会員・予約・会話履歴・ナレッジを保存
- **ダッシュボード** — 管理画面（Next.js / Vercel）
- **Stripe / Square** — オンライン決済（オプション）
- **SwitchBot** — 入退館スマートロック連携（オプション）

### セキュリティ
- JWT + bcrypt による認証
- LINE/Stripe/Square の秘密鍵は一切クライアントに露出しない
- ログから PII（個人識別情報）を自動除去
- タイミング攻撃対策済み
- マルチテナント分離（店舗ごとにデータ隔離）

## 5. 主要機能（実装済み）

### 5.1 LINE AI コンシェルジュ
- お客様のLINEメッセージに Claude Sonnet が自動応答
- @メンションで有人対応に切替
- 会話履歴10往復を記憶（コンテキスト保持）
- Webナレッジ検索（Brave Search、Standard 以上）
- ボット名・トーン・エスカレーション条件を店舗ごとに設定可

### 5.2 AI予約（最短3往復で確定）
```
お客様: 明日の10時空いてる？
  ↓
LINX: 明日10:00〜11:00、田中トレーナーで空きがあります。ご予約しますか？
  ↓
お客様: はい、山田太郎でお願い
  ↓
LINX: ご予約完了しました。当日お気をつけてお越しください。
```
- 空きがない場合は **代替候補3つを自動提示**
- 定員管理（サービス単位・時間枠単位）
- 予約後の自動フロー: 事前フォーム → リマインド → No-show時自動リスケ提案 → 入会案内 → 3日後フォロー

### 5.3 会員管理（/dashboard/customers）
- ステータス4種（体験・会員・休会・退会）
- チェックイン記録
- LINE一斉配信（セグメント別）
- CSV一括取込
- LINE ID と会員情報の自動紐付け

### 5.4 予約管理（/dashboard/reservations）
- Googleカレンダー風の週間グリッドUI
- ドラッグ&ドロップで予約変更
- スタッフ単位・サービス単位でフィルタ
- JST固定、日本の祝祭日表示（2025-2027）

### 5.5 スタッフ管理（/dashboard/staff）
- シフト入力（2ヶ月先まで一括）
- 休みカレンダー（年単位で俯瞰）
- 担当制 / フリー制の切替

### 5.6 メニュー管理（/dashboard/services）
- サービス項目・価格・所要時間・定員
- 時間帯別の料金変動（ピーク/オフピーク）
- LINE Botがそのまま引用して提案

### 5.7 入退館管理（/dashboard/entry）
- SwitchBot連携でスマートロック解錠
- LINEチェックインでタッチレス入館
- エニタイム方式（無人ジム）に対応

### 5.8 ナレッジ管理（/dashboard/knowledge）
- テキスト / PDF ファイルをアップロード
- AI が回答に引用
- 検索結果をダッシュボードで確認

### 5.9 決済（Stripe + Square）
- 月会費の自動課金
- LINEから決済リンクを送付
- Square連携（POS決済の店舗向け）

### 5.10 会話ログ（/dashboard/logs）
- AIが何に対してどう答えたかを全件確認可能
- 品質チェックと運用改善に利用

### 5.11 設定（/dashboard/settings）
- 店舗情報・営業時間・臨時休業（6ヶ月先まで）
- ボット名・トーン・エスカレーション条件
- 決済APIキー・通知設定

### 5.12 マルチ店舗対応（FC本部向け）
- 本部アカウントで全店舗を横断管理
- 店舗ごとの権限フィルタ
- 店舗切替プルダウン

## 6. セットアップ手順（初回・所要30分）

### Step 1 — アカウント作成（5分）
1. https://linx-rouge.vercel.app を開く
2. 「無料ではじめる」 → メールアドレス・パスワード登録
3. 店舗情報（名前・業種・住所）を入力

### Step 2 — LINE公式アカウント連携（10分）
1. LINE Developers Console で「Messaging API Channel」を作成
2. **Channel Access Token** と **Channel Secret** をコピー
3. LINX ダッシュボード /dashboard/setup で2つを貼り付け
4. Webhook URL を LINE Developers に設定（LINXが自動生成したURL）
5. 「疎通確認」ボタンで接続テスト → ✅ 緑チェック

### Step 3 — お店の情報を登録（10分）
1. 営業時間・定休日を設定
2. スタッフを登録
3. メニュー（サービス）を登録
4. ナレッジ（店舗独自の質問回答）をテキスト or PDF で追加

### Step 4 — 試運転（5分）
1. 自分のLINEで店舗の公式アカウントに話しかける
2. AI応答が返ってくることを確認
3. 予約テスト（「明日10時空いてる？」）
4. ダッシュボードの会話ログで挙動確認

**完了** — あとはお客様からのメッセージをAIが24時間受けます。

## 7. ダッシュボードの使い方（日常運用）

### 7.1 毎朝のルーティン（5分以内）
1. **ダッシュボードTOP** — 月間応答数・エラー有無を確認
2. **予約管理** — 今日の予約一覧をチェック、スタッフ配置を確認
3. **会話ログ** — 昨晩の AI応答を10件程度スクロールし品質確認
4. **会員管理** — 新規登録された会員を歓迎（挨拶メッセージを送る）

### 7.2 新しい質問が増えてきたら
- 会話ログで「AIがうまく答えられない質問」を発見
- ナレッジ管理にその回答を追加（テキスト or PDF）
- 次回から自動で答えられるようになる

### 7.3 スタッフ/メニューに変更があったら
- スタッフ管理 → 追加 / シフト更新
- メニュー管理 → 料金変更 / 新メニュー追加
- 即座に AI の回答に反映される

### 7.4 LINE配信（キャンペーン告知など）
- 会員管理 → 一斉配信
- セグメント（会員ステータス別）で絞り込み可能
- 月間応答数は消費しない（配信は別カウント）

## 9. よくある質問（運用FAQ）

| Q | A |
|---|---|
| AIが間違った情報を答えてしまった | ナレッジに正しい情報を追加 → 即反映 |
| お客様が「人と話したい」と言ったら？ | 会話中に「スタッフ」「担当者」等のキーワードが出ると自動エスカレーション、スタッフに通知 |
| 複数店舗を切り替えたい | 本部アカウントなら画面上部の店舗プルダウンから即切替 |
| CSVで既存会員を取り込みたい | 会員管理 → CSVインポート（列フォーマットはテンプレあり） |
| LINE公式アカウントは既存のを使える？ | はい。Channel Access TokenとSecretさえあれば既存のアカウントに機能追加できます |
| プランを途中で変えられる？ | /dashboard/billing で月単位で変更可能 |
| データのバックアップは？ | Railway PostgreSQL の自動バックアップ（日次）+ CSV エクスポートいつでも |
| 月間応答数を超えたら？ | プランアップグレードを促すメッセージ、AIは引き続き応答（追加課金なし） |

## 10. 運用者の注意点

### やってはいけないこと
- ❌ 個人情報（クレカ番号・パスワード等）を LINE で求めない（AIもそう設計してあります）
- ❌ LINX のログイン情報をLINEで配信しない
- ❌ ダッシュボードの「設定」で Channel Secret を他人に見せない

### 推奨すること
- ✅ 週1回、会話ログをざっと見て AI の回答品質をチェック
- ✅ 新メニューや新スタッフはその日のうちにダッシュボードに登録
- ✅ ナレッジは「よくある質問」を月1回見直して追加
- ✅ 月末に月次レポート（応答数・予約数）を見て効果測定

## 11. トラブルシュート

| 症状 | 確認すること |
|---|---|
| LINEで話しかけても返事が来ない | /dashboard/settings で Webhook 疎通状態を確認、OK なら再起動 |
| 予約確定ができない | メニュー登録の「定員」と「営業時間」が正しいか確認 |
| AIがプランを推しすぎる/足りない | /dashboard/settings の「ボットトーン」を調整（ていねい/フレンドリー/プロフェッショナル） |
| ダッシュボードにログインできない | パスワードリセット → それでもダメならLINEでサポート |
| 月間応答数が急増している | 正常に使われている証拠。プラン見直しを検討 |
| カレンダーに祝日が出ない | ブラウザキャッシュ。Cmd+Shift+R でハードリフレッシュ |
